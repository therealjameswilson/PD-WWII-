# Archive Response Tracker

Use this tracker after sending a request from `archive-request-send-queue.md`. Each row records repository acknowledgment, ticket/case number, fees, delivery status, missing material, and downstream register updates.

## Response Workflow

1. Copy the `RESP` row for the request when the request is sent.
2. Fill `Date Sent`, ticket/case number, contact path, fee/invoice status, and estimated response date.
3. When files arrive, save them under `Source Copy Destination` and record delivery URL, file count, covered/missing candidates, page/image issues, and access notes.
4. Update the listed QA, citation, selection, duplicate, annotation, and readiness rows before promoting a document.
