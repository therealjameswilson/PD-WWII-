Document Inclusion Guide

How To Decide What Goes In

Use this guide to move from a declassified candidate document to a defensible FRUS inclusion. The working rule is simple: no document advances until it has a source copy, a duplicate screen, a published-style source note, and an inclusion rationale tied to public diplomacy and foreign policy.

Four checks for every candidate

Keep the guide narrow. For each candidate, answer only these questions before spending time on broader research.

  1. Do we have the actual source copy, not just a catalog or purport-card lead?
  2. Is it absent from already-published FRUS after exact and broad searches?
  3. Can the source note be written to published FRUS standards?
  4. Does the document show foreign-policy public diplomacy, not only domestic morale or folder-level context?

Work Order

First Twenty Work Blocks

1. Freeze the Candidate Register

Open the static 50-candidate register, preserve the exact public FRUS screen for each row, and assign one initial disposition: Promote to source-copy pull, Hold for context, Boundary review, or Exclude.

2. Run the FRUS Duplicate Gate

Start from the exact-screen audit, then save broad topical History.state.gov searches. Record broad result count and final duplicate ruling before any document moves forward.

3. Pull RG 59 Source Copies

Start with the 103.9161 and 103.9166 purport-card batches. Request the underlying Central Decimal File documents, not just the card images, and attach any enclosures.

4. Pair with RG 84 Post Files

Use London, neutral Europe, and Latin America post-file searches to find implementation, clearance, and foreign-reaction evidence that explains why a document belongs in a diplomatic volume.

5. Pull OWI Directive Sets

Build the RG 208 directive packet around long-range policy, special guidance, London/Moscow traffic, OWI-OSS relations, and PWE crosswalks.

6. Check FDR and Boundary Material

Pull White House OWI files and review rumor-control or War Department propaganda material only when it proves foreign-policy significance, presidential control, or State/OWI coordination.

7. Build the Packet Ledger

Every promoted document needs a source-copy packet, selection rationale, duplicate proof, boundary ruling, and cross-reference to policy, implementation, or reaction evidence.

8. Draft the Working Chronology

Use the chronology seed register to order candidates by date, identify editorial role, and keep proof gates visible before assigning final document numbers.

9. Lock the Duplicate Audit

Use the duplicate-gate register to preserve the exact-screen result count already logged, then add broad History.state.gov result counts and final duplicate rulings before promotion.

10. Record the Editorial Decision

Use the selection-decision register to assign promote, hold, boundary, duplicate, or exclude rulings with rationale, reviewer, decision date, source-note status, and proposed FRUS number.

11. Normalize the Source Note

Use the source-note/citation register to preserve the draft source note, flag missing box, folder, item, image, duplicate, and evidence fields, and assign the next citation action.

12. Lock Authority Control

Use the authority-control register to settle preferred person and office names, aliases, title-date checks, organization succession, boundary notes, and annotation/index normalization tasks.

13. Assemble the Manuscript Spine

Use the document-assembly register to group candidates by working FRUS slot, chapter, section, assembly function, proof gate, headnote task, and next action before assigning final numbers.

14. Build Evidence Episodes

Use the episode-evidence crosswalk to pair candidate rows into policy, implementation, reaction, public-line, and boundary packets with missing-evidence and next-pull notes.

15. Execute the Pull Plan

Use the archive-trip pull plan to group source-copy requests by work block, repository path, capture checklist, citation fields, episode need, access check, copy format, and exit criteria.

16. Run Receipt QA

Use the source-copy receipt/QA register to check page order, image completeness, reverse sides, enclosures, routing marks, OCR, transcripts, citation gaps, duplicate proof, and promotion readiness before final selection.

17. Draft Annotations

Use the annotation-drafting register to answer the document's narrative question, normalize persons and offices, tie the source note to citation proof, and list final-text blockers before writing FRUS headnotes or annotations.

18. Re-Triage the Queue

Use the compiler readiness scorecard to sort candidates by work priority, final-readiness score, open gates, main blocker, source-copy tier, and the next 30-minute task.

19. Send Request Queue

Use the archive request send queue to copy the next subject line and request block, attach the named register rows, and record response dates before updating receipt QA and citations.

20. Track Responses

Use the archive response tracker to record ticket numbers, fee status, delivery links, files received, missing candidates, follow-up dates, and every QA, citation, duplicate, selection, annotation, and readiness row that needs an update.

Packet Standard

No Candidate Advances Without These Fields

  • Repository, record group or collection, series, folder, box, NAID, and file number.
  • Document date, author or post, recipient, classification, and routing stamps.
  • Full source copy or image link, including enclosures and marginalia when present.
  • Potential-candidate register row with exact public FRUS screen, source locator, downstream IDs, and next compiler action.
  • Exact FRUS duplicate search and broader History.state.gov topical search.
  • Selection rationale: policy decision, implementation, reaction, handoff, or boundary value.
  • Boundary ruling for military, intelligence, domestic morale, or non-information OCIAA material.
  • Cross-reference to at least one related policy, post, public-line, or reception record.
  • Preferred person and office names, title-date evidence, aliases, and organization succession checked against the authority register.
  • Working chapter, section, document slot, assembly function, headnote task, and next action recorded before final numbering.
  • Episode-level policy, implementation, reaction, public-line, boundary, and missing-evidence crosswalk completed before drafting narrative headnotes.
  • Archive pull-plan row with repository path, capture checklist, citation fields, access/restriction check, copy format, and exit criteria.
  • Receipt-QA row confirming image completeness, page order, reverse sides, enclosures, routing marks, OCR/transcript status, citation gaps, duplicate proof, authority checks, and promotion readiness.
  • Annotation-drafting row with narrative question, headnote task, authority forms, citation dependency, episode cross-references, and final-text blockers.
  • Readiness-scorecard row with work-priority score, final-readiness score, open gates, main blocker, source-copy tier, and next task.
  • Request-send row with send rank, subject line, request channel, tier mix, attachment rows, next archive action, and registers to update after response.
  • Response-tracker row with ticket/case number, fee/payment status, delivery URL, file count, covered/missing candidates, page/image issues, and follow-up date.
  • Named reviewer, decision date, and rationale for every promoted, held, boundary-review, or excluded row.

Presidential and White House Control

Pull President's Secretary's File material for Office of War Information, confidential OWI files, and Survey of Intelligence folders. Extract only documents that show FDR action, White House routing, or connection to State/OWI decisions.

This is the check against overbuilding the volume from agency files alone.

Decision Gate

Promotion Rules

  1. Promote policy decisions, presidential control, State-post implementation, foreign reaction, and handoff records only after the source-copy and duplicate evidence can be inspected.
  2. Hold records that are useful context but lack source-copy proof or duplicate checks.
  3. Boundary-review domestic morale, OSS, War Department psychological warfare, and non-information OCIAA material.
  4. Exclude purely tactical, purely domestic, duplicate, or folder-level records once stronger individual documents are identified.
